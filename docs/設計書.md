# MenuBarDockX 設計書

> 本書はソースコードからリバースエンジニアリングして作成した設計ドキュメントです。
> 対象バージョン: **v1.3.0** / 対象 OS: macOS 14 (Sonoma) 以降 / 言語: Swift 5・AppKit

---

## 1. 目的と背景

macOS のメニューバーには、サードパーティ製ステータスアイコンが増えると右端から
あふれて見えなくなる問題がある。ノッチ搭載 Mac ではノッチ裏の **dead zone** に
アイコンが押し込まれ完全に不可視になる。macOS にはこれを救済する公式の
オーバーフロー UI が存在しない（Control Center はシステム機能専用）。

MenuBarDockX は、

1. 隠れた（ノッチ裏・画面外の）ステータスアイテムを **AX (Accessibility) API** で検出し、
2. 自身の NSStatusItem アイコンを **▾** に切り替えて「隠れているアイテムがある」ことを通知し、
3. クリックまたはグローバルショートカットで **フローティングパネル** に集約表示・操作可能にする

ことを目的とする。すべてローカルで完結し、ネットワーク通信・テレメトリは一切行わない。

```
           │               └──────────────────────────────────────┘
           ▼
   ┌──────────────────────┐   ┌──────────────┐   ┌──────────────┐
   │ ClassificationRules   │   │  DataStore   │   │  Category /  │
   │ Manager (bundle→cat)  │   │ (JSON/UD永続)│   │ MenuBarItem  │
   └──────────────────────┘   └──────────────┘   └──────────────┘
```

UI ランタイムは **AppKit のみ**（NSStatusItem + NSPanel + NSView）で構成する。
SwiftUI は `@main` のエントリポイント（`MenuBarDockXApp`）が空の `Settings` シーンを
保持するためだけに使い、実 UI には一切用いない。

> **設計判断（重要）**: 初期実装では SwiftUI `MenuBarExtra` の `isInserted` バインディングで
> ▾ を出し入れしていたが、`@Published` 変化 → SwiftUI 再描画 → NSStatusItem 生成/破棄 →
> AX 検出変化 → 再び `@Published` 変化、という無限ループで暴走しメモリを使い果たす問題が
> あった。これを根絶するため MenuBarExtra を全廃し、**既存 NSStatusItem のアイコンを
> 差し替える方式**に切り替えた。

## 4. 起動シーケンス（`applicationDidFinishLaunching`）

1. `NSApp.setActivationPolicy(.accessory)` — Dock アイコンを出さない常駐アプリ化。
2. 同一バンドル ID の重複インスタンスを `terminate()`（開発時の二重起動対策）。
3. `promptLoginItemIfNeeded()` — 未登録かつ未提示なら、初回のみログイン自動起動を提案。
4. `EnvironmentChecker.run()` → AX 権限・OS バージョン・Rosetta を確認し、必要なら権限要求。
5. `setupStatusItem()` — NSStatusItem を生成、通常アイコンを設定、メニューを構築。
   併せて `OverflowStatusManager.onOverflowModeChanged` コールバックを登録。
6. `OverflowStatusManager.shared.start(...)` — ポーリング開始。
7. `GlobalShortcutManager.shared.start()` — グローバルショートカット監視開始。
8. 言語変更通知を購読してメニューを再構築。

## 5. オーバーフロー検出ロジック

### 5.1 ノッチ検出 `NotchDetector`

`NSScreen.auxiliaryTopLeftArea` / `auxiliaryTopRightArea`（macOS 12+）から
ノッチ左右端の X 座標を算出する。

```
leftEdgeX  = screen.frame.minX + auxiliaryTopLeftArea.maxX
rightEdgeX = screen.frame.minX + auxiliaryTopRightArea.minX
hasNotch   = rightEdgeX > leftEdgeX
```

DEBUG ビルドでは擬似ノッチ（画面中央 200pt）に差し替え可能。
ノッチ非搭載 Mac では `hasNotch = false` となり、隠れ判定は画面右端基準にフォールバックする。
**この計算は画面サイズに依存しないため 14 / 16 インチや外部ディスプレイでもそのまま機能する。**

### 5.2 隠れアイテム判定 `MenuBarEnumerator.enumerateHiddenItems`

各アプリの AX 要素を走査し、アイコンの **右端**がノッチ右端より左にあるものを「隠れている」と判定する。

```
iconRight     = pos.x + size.width
isInNotchZone = hasNotch && pos.x > 0 && iconRight < rightEdgeX
isHidden      = isHiddenLeft || isInNotchZone
```

> **設計判断**: v1.2.0 までは「左端 + 40pt マージン」で判定していたが、ノッチ境界をまたぐ
> アイコン（左端は隠れているが右端は可視）が可視領域とパネルの両方に二重表示される
> バグがあった。v1.3.0 で**右端基準**に変更して解消した。

### 5.3 自分自身の dead zone 判定 `OverflowStatusManager.poll`

MenuBarDockX の NSStatusItem 自身がノッチ裏に押し込まれると ▾ が見えなくなる。
自分のウィンドウ X 座標で判定する（他アプリ判定と異なりマージンは使わない）。

```
selfInDeadZone = ownX < 10
              || (hasNotch && ownX > 0 && ownX < rightEdgeX)
```

`selfInDeadZone` が真なら、たとえ他アプリの隠れアイテムが 0 でもオーバーフロー扱いとし、
▾ 状態を維持する。可視化はグローバルショートカット ⌃⌥⌘M で担保する。

### 5.4 状態遷移 `OverflowStatusManager.update`

```
effectivelyOverflow = !hiddenItems.isEmpty || selfInDeadZone
stateChanged        = (隠れアイテム ID 集合の変化) || (selfInDeadZone の変化)
```

`stateChanged` のときのみ処理する（無駄な再描画と前述の暴走ループを抑止）。
- overflow あり → `enterOverflowMode()`（アイコンを ▾ に）＋ `panel.setItems()`
- overflow なし → `exitOverflowMode()`（通常アイコンに復元）＋ パネルを閉じる

ポーリングは 3 秒間隔。加えて `NSWorkspace.didTerminateApplicationNotification` を購読し、
アプリ終了時は 0.5 秒後に即時 `forceRefresh()` してアイコンを除去する。

## 6. パネル（`OverflowPanelController` / `OverflowPanelContent`）

### 6.1 ウィンドウ構成

- `OverflowPanel`（`NSPanel`, `.borderless`）。`canBecomeKey/Main` を override して true。
- レベルは `statusWindow`。`hidesOnDeactivate = false` で表示制御を完全に自前管理。
- `backgroundColor` は `alpha=0.001` の擬似透明。完全な透明（alpha=0）にすると
  WindowServer がクリックを背後へ通過させてしまうため、ごく僅かに不透明にして
  クリック透過を防止する。

### 6.2 配置 `positionPanel`

アンカー（▾ ボタンのウィンドウ）が可視ゾーンにあればその中央真下に配置。
dead zone にある、またはショートカット起動でアンカー不在の場合は **ノッチ右端の直後**に配置する。
最終的に画面端からはみ出さないようクランプする。Y はメニューバー高さ直下。

### 6.3 表示/非表示のレース対策

`showPanel`/`hidePanel` は `panelGeneration` カウンターで世代管理する。
`hidePanel` のフェードアウト完了ハンドラ発火時に世代が変わっていれば（＝間に showPanel が
割り込んだ場合）`orderOut` をスキップし、パネルが誤って消えるのを防ぐ。

### 6.4 イベント監視

表示中はグローバル/ローカルのマウス・キー監視を張る。
- パネル外の左クリック → 閉じる
- パネル内でも歯車/タブ/アイコン行以外のクリック → 閉じる
- Esc → 閉じる

### 6.5 表示モード

- **Flat**: 全アイコンを 1 行表示（`OverflowNormalView`）。
- **Category**: 上部タブ（`OverflowTabBar`）＋選択カテゴリのアイコン行（`OverflowTabPanelView`）。
  タブは幅が収まらなければ 2 行に折り返す。組み込み 6 カテゴリは常時表示。

### 6.6 編集モード

歯車 → 設定ポップオーバー →「編集モード」で ON。`EditDragOverlay` が
`window.nextEvent` の同期ループでドラッグを処理し、
- アイコン行内ドロップ → 並び替え（`sortOrder` 更新）
- タブ上ドロップ → カテゴリ割当

いずれも DTO を `DataStore` に保存して再起動後も保持する。

## 7. クリック転送

パネル内アイコンのクリックは対象アプリの AX 要素へ転送する。

- **左クリック**: `AXUIElementPerformAction(element, kAXPressAction)`。失敗時は `AXShowMenu` を試行。
- **右クリック**: `AXShowMenu` を button → AXParent → AXWindow の順にフォールバック。
  すべて失敗した場合は MenuBarDockX 独自の簡易メニュー（Toggle / Quit）を表示する。

> **既知の制約**: macOS 15 以降では hidden な `NSStatusBarWindow` が layer=0 に降格され、
> CGEvent による右クリック合成は ControlCenter(layer=25) に横取りされる。
> このため CGEvent 合成方式は採用せず、AX ベース + 簡易メニューのフォールバックチェーンを使う。

## 8. 背景色エンジン `MenuBarGradientSampler`

パネル背景をメニューバーの実際の色に追従させる。

```
① CGWindowListCreateImage でメニューバー全幅をキャプチャ
② 中央 1px 行を切り出し（横方向の色分布を保持）
③ CIGaussianBlur(radius=240) で虹色縞を平滑化（clampedToExtent で端部暗化を防止）
④ CIColorControls で明度(×1.10)・彩度(×0.88)を補正（vibrancy 近似）
⑤ solidOverlay.layer.contentsRect でパネル X 位置に対応するスライスのみ描画
```

表示中は 1 秒間隔で再サンプリングし、ライト/ダーク切替・壁紙変更に追従する。
スクリーン録画権限がない場合はフォールバック色になる（アプリ自体は動作する）。

## 9. 自動分類 `ClassificationRulesManager`

`classification_rules.json`（バンドル同梱）＋ユーザールール（Application Support）を読み込み、
完全一致 → プレフィックス一致の順で bundleID をカテゴリ名に解決する。

```
match(bundleID, rules, prefixRules):
    完全一致ルールがあればそのカテゴリ名
    なければプレフィックス一致ルールのカテゴリ名
    どちらもなければ nil
categoryID(forBundleID:):
    match() の結果（"開発" 等）を Category.presets の固定 UUID にマップ
```

`enumerateHiddenItems` は、保存済み DTO にカテゴリがあればそれを最優先し、
なければ `categoryID(forBundleID:)` で自動割当する。解決できなければ未分類（nil）。

照合ロジックはバンドル/ファイルシステムに依存しない純粋関数 `match(...)` に切り出してあり、
ユニットテスト可能。

## 10. 永続化 `DataStore`

| 対象 | 保存先 |
|------|--------|
| カテゴリ | `~/Library/Application Support/MenuBarDockX/categories.json` |
| アイテム DTO（categoryID, sortOrder） | `.../MenuBarDockX/items.json` |
| 表示設定 `OverflowSettings` | UserDefaults キー `overflowSettings`（JSON） |
| 言語 | UserDefaults キー `appLanguage` |

### スキーママイグレーション

2 軸で管理する。
- 汎用 `schemaVersion`（現行 v5）: `migrateIfNeeded()` が起動時に 1 度実行。v5 で
  `blendBrightness=1.10 / blendSaturation=0.88` に上書き。
- `overflowSettingsSchemaVersion`（現行 v4）: `panelOpacity` の意味変更に追随
  （NSVisualEffectView 廃止に伴い 1.0 = サンプリング色ソリッドへ）。

`OverflowSettings` は `decodeIfPresent` で各フィールドにデフォルトを補うため、
フィールド追加のたびにスキーマを上げずに後方互換を保てる。

## 11. グローバルショートカット `GlobalShortcutManager`

`NSEvent.addGlobalMonitorForEvents(matching: .keyDown)` で keyCode=46(M) かつ
修飾キー `[.control, .option, .command]` を監視し、一致で `togglePanel()` を呼ぶ。
アクセシビリティ権限が必要（本体機能と共通）。3 修飾キーで他アプリとの競合を最小化している。

---

## 12. ログイン項目 `LoginItemManager`

`SMAppService.mainApp.register()/unregister()`（macOS 13+）で管理。
**システム設定 → 一般 → ログイン項目** に表示される。
Sequoia / Tahoe では起動順が早いほどメニューバー可視ゾーンのスロットを確保しやすいため、
初回起動時に自動起動を提案する。

---

## 13. 環境チェック `EnvironmentChecker`

- アクセシビリティ: `AXIsProcessTrusted()`、未許可なら `AXIsProcessTrustedWithOptions` で要求。
- OS バージョン: macOS 14 以降をサポート。
- Rosetta: `sysctl.proc_translated` で判定（フォールバックで `pgrep oahd`）。
- 不備があれば警告ダイアログを表示し、システム設定への導線を提供する。

## 14. デバッグ機能（`#if DEBUG` 限定）

## 16. プライバシー・セキュリティ方針

- ネットワーク通信コードを持たない（完全ローカル）。
- テレメトリ・アカウント・サブスクリプションなし。
- 第三者ライブラリ依存ゼロ（Apple 純正 API のみ）→ 依存由来の脆弱性面を持たない。
- App Sandbox は AX API のために無効。理由を含め全コードを公開し検証可能。
