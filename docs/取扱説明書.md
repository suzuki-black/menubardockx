# MenuBarDockX 取扱説明書

> 対象バージョン: **v1.3.0** / 対応 OS: macOS 14 (Sonoma) 以降（Apple Silicon / Intel）

---

## 1. MenuBarDockX とは

メニューバーに常駐アプリが増えると、アイコンが右端からあふれて見えなくなります。
ノッチ搭載 MacBook では、ノッチの裏にアイコンが押し込まれて完全に見えなくなることもあります。

**MenuBarDockX** は、こうして隠れてしまったメニューバーアイコンを検出し、
**フローティングパネル**にまとめて表示・操作できるようにするユーティリティです。

- 隠れたアイコンがあると、MenuBarDockX のアイコンが **▾**（下向き三角）に変わります。
- ▾ をクリックするとパネルが開き、隠れたアイコンを直接クリックできます。
- すべてローカルで動作し、外部送信・課金・アカウントは一切ありません。

---

## 2. インストール

### 現状：インストーラは準備中

コードサイニング・公証済みの DMG 配布は準備中です。当面はソースからビルドしてください。

### ソースからビルドする

```bash
git clone https://github.com/suzuki-black/MenuBarDockX.git
cd MenuBarDockX
xcodebuild -project MenuBarDockX.xcodeproj \
           -scheme MenuBarDockX \
           -configuration Release \
           CONFIGURATION_BUILD_DIR=./build/Release
open ./build/Release/MenuBarDockX.app
```

初回起動時に Gatekeeper の警告が出た場合は、
**システム設定 → プライバシーとセキュリティ** で「このまま開く」を選びます。

## 3. 初回起動と権限設定

### 3.1 アクセシビリティ権限（必須）

メニューバーアイコンの読み取り・クリック操作、およびグローバルショートカットに必要です。

1. MenuBarDockX を起動する（初回は自動でダイアログが表示されます）。
2. **システム設定 → プライバシーとセキュリティ → アクセシビリティ** を開く。
3. リストの **MenuBarDockX** をオンにする。
4. MenuBarDockX を再起動する。

### 3.2 画面収録権限（任意）

パネル背景をメニューバーと同じ色になじませるために使います（`CGWindowListCreateImage`）。
許可しなくても動作しますが、背景はフォールバック色になります。

**システム設定 → プライバシーとセキュリティ → 画面収録** で MenuBarDockX をオン。

### 3.3 ログイン時の自動起動（推奨）

初回起動時に「ログイン時に自動起動しますか？」というダイアログが表示されます。

macOS Sequoia / Tahoe では、**起動が早いアプリほどメニューバーの見える位置を確保しやすい**
仕組みになっています。自動起動を有効にすると MenuBarDockX が早く起動し、
▾ インジケーターが見える位置に出やすくなります。

後からでも、メニューの **「ログイン時に自動起動」** で切り替えられます
（チェックが付いていれば有効）。

## 4. 基本的な使い方

### 4.1 アイコンの見方

| 表示 | 意味 |
|------|------|
| 通常アイコン（□ 型） | 隠れているアイコンはありません |
| **▾**（下向き三角） | 隠れているアイコンがあります／自身がノッチ裏に入っています |

### 4.2 パネルを開く

- **▾ を左クリック** → オーバーフローパネルが開きます。
- パネル内のアイコンを **左クリック** → そのアプリのアイコンを左クリックしたのと同じ動作。
- パネル内のアイコンを **右クリック** → そのアプリのメニュー（出せない場合は Toggle / Quit の簡易メニュー）。

### 4.3 設定メニューを開く

- 通常時（□ アイコン）: 左クリックでメニュー（About / ログイン自動起動 / 終了）。
- オーバーフロー時（▾）: 左クリックはパネルを開きます。設定メニューは **▾ を右クリック**。

---

## 5. ショートカット（▾ が見えないときの切り札）

```
⌃ + ⌥ + ⌘ + M   （Control + Option + Command + M）
```

どのアプリを使っていても、いつでもオーバーフローパネルを開閉できます。

**▾ がノッチに隠れて見えない場合でも使えます。** メニューバーが混雑して MenuBarDockX の
アイコン自体が押し込まれてしまったときの、確実な代替手段です（アクセシビリティ権限が必要）。

---

## 6. パネルの設定

パネル右上の **歯車アイコン** をクリックすると設定が開きます。

| 設定項目 | 既定値 | 範囲 | 説明 |
|----------|--------|------|------|
| 編集モード | OFF | — | ドラッグ並び替え・カテゴリ変更を有効化 |
| 表示モード | フラット | フラット / カテゴリ | フラット＝1 行表示／カテゴリ＝タブ分類表示 |
| アイコン幅 | 40pt | 32〜48pt（2pt 刻み） | アイコンセルの幅 |
| 明度補正 | ×1.10 | 0.80〜1.20 | 背景色の明るさ |
| 彩度補正 | ×0.88 | 0.70〜1.10 | 背景色の鮮やかさ |
| 背景の不透明度 | 1.0 | 0.0〜1.0 | 0.0＝透明／1.0＝サンプリング色で塗りつぶし |
| クリック後にパネルを閉じる | ON | — | アイコンクリック後に自動で閉じる |
| 言語 | — | English / 日本語 | 表示言語の切替 |

## 7. カテゴリと編集モード

### 7.1 自動分類

よく使うアプリ（Docker・Dropbox・1Password・Raycast など）は、
バンドル ID に基づき自動的にカテゴリ（開発／クラウド／セキュリティ／ユーティリティ等）へ
振り分けられます。「カテゴリ」表示モードに切り替えるとタブで分類表示されます。

### 7.2 手動でカテゴリ・並び順を変える（編集モード）

1. ▾ を左クリックしてパネルを開く。
2. 歯車 → 設定を開く。
3. **「✏️ 編集モード」** ボタンを押す（「✏️ 編集中」表示で ON）。

編集モード中の操作：

| 操作 | 効果 |
|------|------|
| アイコンをドラッグ（行内） | 並び順を変更 |
| アイコンをタブへドラッグ | そのカテゴリに割り当て |
| アイコンを右クリック | カテゴリ選択メニューを表示 |

並び順・カテゴリは再起動後も保持されます
（`~/Library/Application Support/MenuBarDockX/items.json`）。

もう一度「編集モード」ボタンを押すと終了します。

---

## 8. 終了方法

- 通常時: アイコンを左クリック →「MenuBarDockX を終了」（`⌘Q`）。
- オーバーフロー時: ▾ を右クリック →「MenuBarDockX を終了」。

## 9. トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| アイコンが表示されない | アクセシビリティ権限が未許可。第 3.1 節の手順で許可し再起動。 |
| ▾ が見当たらない | MenuBarDockX 自身がノッチ裏に押し込まれています。**⌃⌥⌘M** でパネルを開けます。ログイン自動起動を有効にすると改善します。 |
| パネルの背景色がメニューバーと合わない | 画面収録権限が未許可（第 3.2 節）。または設定で明度/彩度/不透明度を調整。 |
| 同じアプリのアイコンが複数出る | Stats など、用途別に複数のステータスアイテムを登録するアプリです（CPU・ネットワーク等）。各アイコンは個別にクリックできます。 |
| アイコンが汎用アプリアイコンになる | そのアプリが AX 経由で画像も説明も公開していないためです（仕様上の制約）。 |
| 右クリックでアプリのメニューが出ない | macOS 15 以降では一部アプリのメニューを呼び出せず、簡易メニュー（Toggle / Quit）にフォールバックします。 |

---

## 10. プライバシー

MenuBarDockX は取得した情報を **一切外部に送信しません**。
ネットワーク通信コードを持たず、テレメトリ・アカウント・サブスクリプションもありません。
すべての処理はあなたの Mac の中だけで完結します。

> Intel Mac・macOS 14 (Sonoma) は未検証です。
